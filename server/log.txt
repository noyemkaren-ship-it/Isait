Вы сказали: "Как брать все логи из Express"Для сбора всех логов (как системных событий Express, так и входящих HTTP-запросов) лучше всего использовать модуль morgan для запросов и стандартную переменную DEBUG для внутренних процессов.1. Логирование входящих HTTP-запросов (Morgan)Самый популярный способ логировать все HTTP-запросы — использовать промежуточное ПО morgan.Установите пакет:bashnpm install morgan
Используйте код с осторожностью.Добавьте его в код вашего сервера:javascriptconst express = require('express');
const morgan = require('morgan');
const app = express();

// Логирует все запросы в формате 'combined' (стандартный формат логов Apache)
app.use(morgan('combined'));

app.get('/', (req, res) => {
  res.send('Привет, мир!');
});

app.listen(3000);
Используйте код с осторожностью.2. Запись логов в файл (Morgan + File Stream)Если вам нужно не просто выводить логи в консоль, а сохранять их в текстовый файл:javascriptconst express = require('express');
const morgan = require('morgan');
const fs = require('fs');
const path = require('path');
const app = express();

// Создаем поток записи для файла access.log
const accessLogStream = fs.createWriteStream(path.join(__dirname, 'access.log'), { flags: 'a' });

// Подключаем morgan для записи в файл
app.use(morgan('combined', { stream: accessLogStream }));
Используйте код с осторожностью.3. Получение внутренних логов самого ExpressСам фреймворк Express не выводит все свои внутренние процессы по умолчанию для избежания лишнего шума. Чтобы включить полную отладку, используйте переменную окружения DEBUG=express:* при запуске приложения.В Linux / macOS:bashDEBUG=express:* node server.js
Используйте код с осторожностью.В Windows (PowerShell):powershell$env:DEBUG="express:*"; node server.js
Используйте код с осторожностью.В Windows (Command Prompt):cmdset DEBUG=express:*
